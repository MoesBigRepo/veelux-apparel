# Veelux Apparel — SEO Report

Approach: `seo-core` checklist applied to the React rebuild. Every item below is **implemented in code**
(component or generator) rather than left as a recommendation.

## On-page / metadata
| Item | Before (live Shopify) | After (rebuild) |
|------|----------------------|-----------------|
| Per-page `<title>` | Generic / theme default | Unique, keyword-mapped via `Seo` component (`src/components/Seo.tsx`); home = `VEELUX — All Eyes on You`, PDP = `{product} · VEELUX`, etc. |
| Meta description | Thin / missing | Unique per product, collection, page (≤160 chars). |
| Canonical | — | `<link rel=canonical>` on every route. |
| OG / Twitter cards | Minimal | Full `og:*` + `twitter:summary_large_image` per page, image = product hero. |
| Heading hierarchy | Marquee-heavy, weak | One `<h1>` per page, semantic `<h2>` sections. |
| Image alt text | `alt="index"`, missing | Descriptive alt on every image (`{title} — view N`, collection/brand alts). |

## Structured data (JSON-LD) — `src/components/Seo.tsx`
- **Product** + **AggregateOffer** (price range, currency, availability, SKU) on every PDP.
- **BreadcrumbList** on PDP + collections.
- **Organization** (Brooklyn, email, logo) + **WebSite** on home.

## Technical SEO
- `sitemap.xml` — 28 URLs (home, collections index, 12 collections, 10 products, about, contact, policies), generated by `scripts/gen_seo.mjs`.
- `robots.txt` — allows all, points to sitemap.
- Clean URL structure mirrors Shopify (`/products/{handle}`, `/collections/{handle}`) so external links/SEO equity map 1:1.
- `theme-color`, `lang=en`, viewport set.

## Core Web Vitals
- **LCP**: hero poster image is the LCP element; Seedance video is `preload="none"` and overlays after, never blocking. Above-fold product images `loading="eager"`, rest `lazy`.
- **CLS**: every image sits in a fixed `aspect-[4/5]` / `aspect-[3/4]` box → zero layout shift.
- **JS**: route-level code-splitting (`React.lazy`) — home ships without PDP/collection/about code; framer-motion shared chunk cached across routes.
- **Fonts**: `@fontsource` variable fonts, subset per-script by Vite, `woff2`, self-hosted (no render-blocking Google Fonts request — also fixes the original external-font dependency).

## Residual / deploy-time
- `og:image` currently brand logo for non-product pages; a per-page social card generator can be added.
- On deploy, submit `sitemap.xml` to Search Console and set the canonical host to the production domain (currently canonical points at `veeluxapparel.com`; update `SITE.url` in `src/lib/data.ts` if hosting elsewhere).
- SPA routing needs a host rewrite (`/* -> /index.html`) — include `vercel.json` / `_redirects` at deploy.
